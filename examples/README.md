# Examples

This directory contains lightweight example inputs to help users understand how to organize requirement materials.

These examples are not meant to be production-complete datasets. They are small reference sets for onboarding, demos, and regression thinking.

## Included Examples

- `auth-basic/`
  Basic login and registration requirement example.
- `payment-refund/`
  Payment and refund flow example.
- `reporting-dashboard/`
  Reporting and dashboard example with custom vocabulary potential.

## Suggested Usage

1. Read one example folder.
2. Compare its files with the `inputs/` structure.
3. Copy the pattern when preparing your own requirement materials.
4. If your domain vocabulary is different, extend the extractor using `extractor-overrides.json`.
