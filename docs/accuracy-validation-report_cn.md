# 准确度验证报告

这份文档汇总了平台在不同类型产品需求上的准确度验证结果，用于说明当前工具链在哪些场景更稳、哪些场景仍需要人工复核。

## 验证范围

本轮基线样例覆盖：

- 登录注册
- 支付退款
- 报表看板
- 审批流
- 工单生命周期

验证重点包括：

- 页面、动作、规则、流转是否能被正确抽取
- `merged-dsl.json` 是否具备可继续生成的结构完整性
- `validation-report.md` 是否能暴露高风险项
- 下游 PRD、OpenSpec、测试稿、接口稿是否仍然保持需求语义

## 结论摘要

### 表现较稳的场景

- 页面显式、流程显式、规则显式的需求
- 有 PRD + notes + context 组合输入的需求
- 主流程清晰、异常路径数量有限的需求

### 风险较高的场景

- 页面名高度相似的结果页或状态页
- 主要依赖截图推断、文本说明较弱的需求
- 中间状态很多、条件分支较密的审批流和状态机需求

## 当前能力判断

- 可行性：高。主流水线可稳定运行，结构化产物可持续生成。
- 完整性：中高。对常见产品需求已能形成统一 DSL、规格稿和执行衔接产物。
- 准确度：中高。显式需求更稳，弱证据场景需要更多人工复核。
- 可信度：明显提升。证据、可信度摘要和 validation 已能帮助用户判断结果边界。

## 后续优化重点

1. 继续增强 validation 的语义风险识别能力
2. 继续补充复杂审批流、工单流、状态机类样例
3. 继续提升截图、流程图与文本证据之间的对齐能力
4. 继续加强低可信度项的显式提示与复核建议
