# 一、用法
inputs/ 是“需求原始输入池”，是整个 AI 流程的唯一入口。
它的作用不是存文件，而是： 把所有分散的需求信息集中到一个“可被 AI 扫描的上下文空间”。

screenshots  -> 看到页面长什么样
prd          -> 知道业务在做什么
notes        -> 知道真实怎么做（补坑）
context      -> 知道系统怎么约束



# 二、为什么必须分 4 个子目录（不是随便分的）
本质是： 多页面 + 多文档 + 逻辑穿插 + 信息碎片；所以 inputs 不能乱放，必须分层。

## 1、screenshots/（视觉信息）
### 放什么：
墨刀页面截图
弹窗截图
流程图
交互图
UI 原型

### 作用：
👉 提供“可见信息”
页面结构
按钮
文案
状态展示
⚠️ 但注意：
它不能提供完整逻辑

## 2、prd/（主需求文档）

### 放什么：
PRD.md
Word 转 md
需求说明
产品文档

### 作用：
👉 提供“业务语义”

### 规则
流程描述
功能说明
## 3、notes/（补充信息）

### 放什么：
临时说明
产品口头补充
IM/飞书记录整理
会议纪要

### 作用：
👉 补“PRD 没写但真实存在的逻辑”
这是最容易被忽略的，但实际最重要。

## 4、context/（系统上下文）

### 放什么：
接口文档
权限说明
角色定义
埋点规则
历史版本说明
业务规则库

### 作用：
👉 提供“系统级依赖和约束”
比如：
登录态
权限控制
token机制
服务依赖
